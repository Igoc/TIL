## Git

프로젝트의 버전 관리를 위한 도구이다.

### 특징

- 파일의 버전을 관리할 수 있다.
- 작업 단위를 나누어 관리할 수 있다.
- 누가, 언제, 어떤 부분을 수정했는지를 손쉽게 파악할 수 있기 때문에, 협업해서 프로젝트를 수행하기에 유용하다.

## GitHub

Git 원격 저장소와 Git을 이용한 커뮤니티 기능이 합쳐진 서비스이다. 이러한 서비스를 제공하는 것은 GitHub 이외에도 GitLab, Bitbucket 등이 존재한다.

## Git 초기화

`git init` 명령어를 이용하여 현재 프로젝트를 Git 프로젝트로 설정할 수 있다.

### 기본 브랜치명 변경

``` bash
# Git 프로젝트 생성 시 기본 브랜치명을 변경
git config --global init.defaultBranch <브랜치명>

# 설정한 기본 브랜치명 확인
git config --get init.defaultBranch
```

## 버전 관리와 Commit

Git에서는 버전 관리를 위해 버전마다 파일을 따로 만들어줄 필요 없이, Git을 이용해 중간중간 현재 프로젝트의 상태를 저장해주기만 하면 된다.

### 버전 관리

누가, 언제, 어떤 부분을 수정했는지에 대한 정보를 남겨서, 파일의 변경 기록을 관리하는 것을 의미한다.

### Staging

Commit에 반영할 파일을 선택하는 작업을 의미한다.

``` bash
git add <파일명>
```

#### Staging 상태 확인 명령어

``` bash
git status
```

### Commit

현재 프로젝트의 상태를 저장하는 작업을 의미하며, Git은 이전 Commit 기록을 알고 있기 때문에 자동으로 어떤 부분이 바뀌었는지를 알려준다.

``` bash
git commit -m <Commit 메시지>
```

#### Commit 로그 확인 명령어

``` bash
git log
```

## 원격 저장소 사용하기

로컬 저장소는 내 컴퓨터에 저장되어 있는 Repository를 의미하고, 원격 저장소는 GitHub과 같이 다른 곳에서 접속할 수 있는 공간에 저장되어 있는 Repository를 의미한다.

``` bash
# 원격 저장소 복제
git clone <원격 저장소 URL>

# 로컬 저장소에 원격 저장소 연결
git remote add <원격 저장소명> <원격 저장소 URL>
```

### Push

로컬 저장소의 Commit들을 원격 저장소에 반영하는 작업을 의미한다.

``` bash
git push <원격 저장소명> <브랜치명>
```

### Pull

원격 저장소의 Commit들을 로컬 저장소에 반영하는 작업을 의미한다.

``` bash
git pull <원격 저장소명> <브랜치명>
```

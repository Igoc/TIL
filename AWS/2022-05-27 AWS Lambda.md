## AWS Lambda

AWS에서 제공하는 이벤트 중심의 서버리스 컴퓨팅 서비스이다.

### 장점

- 필요할 때만 함수가 호출되고 비용이 부과되는 방식이기 때문에 비용을 절감할 수 있다.
- 인프라를 따로 관리할 필요가 없기 때문에, 빠르게 서버를 구성할 수 있다.

### 단점

- 메모리와 같은 리소스의 최대 크기가 제한되어 있다.
- 오랜만에 실행할 경우 딜레이(Cold Start)가 발생한다.

### 구성

#### 트리거

특정 AWS 리소스에서 어떤 이벤트가 발생할 때마다 자동으로 함수가 호출되도록 설정할 수 있는 기능이다.

#### 권한

특정 AWS 리소스에 접근할 수 있는 권한을 설정할 수 있는 기능이다.

#### 대상

함수가 성공 또는 실패하고 난 후 다른 대상이 호출되도록 설정할 수 있는 기능이다.

#### 함수 URL

함수를 호출하기 위한 URL을 간단하게 구성할 수 있는 기능이다.

#### 환경 변수

함수 내에서 사용할 환경 변수를 정의할 수 있는 기능이다.

#### VPC

해당 함수가 어떤 VPC에 포함될지 설정할 수 있는 기능으로, AWS Lambda는 VPC를 설정할 경우 아웃바운드가 자동으로 막히기 때문에 외부 통신이 필요하다면 별도로 설정을 해주어야 한다.

#### 코드 서명

코드에 서명을 적용하여 확인되지 않은 코드의 배포를 제한할 수 있는 기능이다.

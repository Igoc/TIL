## AWS VPC

논리적으로 격리된 가상 네트워크를 이용하여 AWS 리소스를 분리할 수 있게 만들어주는 서비스이다.

### 서브넷(Subnet)

AWS VPC에서 관리하는 IP 주소를 더 작은 단위로 나누어 둔 것으로, 인터넷 통신이 허용된 퍼블릭 서브넷과 내부 통신만 사용하는 프라이빗 서브넷으로 나누어 관리할 수 있다.

### 라우팅 테이블(Routing Table)

네트워크 트래픽을 전달할 위치를 결정하는 데 사용하는 라우팅 정보를 관리하는 테이블로, 트래픽을 전달할 서브넷과 트래픽을 전송할 게이트웨이를 지정할 수 있다.

### 인터넷 게이트웨이(Internet Gateway)

AWS VPC의 리소스와 인터넷 간의 통신을 활성화하기 위해, AWS VPC에 연결하는 게이트웨이이다.

### NAT 게이트웨이(NAT Gateway)

프라이빗 서브넷에 포함된 AWS 리소스를 인터넷, 온프레미스 네트워크, 다른 AWS VPC에 연결하기 위한 게이트웨이이다.

### 탄력적 IP

특정 AWS 리소스에 고정된 IP를 부여할 수 있는 기능이다.

### 보안 그룹

방화벽과 비슷한 역할을 수행하는 기능으로, 규칙을 설정하여 보안 그룹에 연결된 AWS 리소스로의 접근을 통제한다.

#### 인바운드 규칙

외부에서 AWS 리소스로 접근하는 것을 통제하기 위한 규칙을 설정할 수 있다.

#### 아웃바운드 규칙

AWS 리소스에서 외부 리소스로 접근하는 것을 통제하기 위한 규칙을 설정할 수 있다.

## AWS ELB

트래픽을 자동으로 여러 AWS EC2 인스턴스에 분산시켜주는 서비스이다.

### 이점

서비스에 대한 요청을 자동으로 여러 AWS EC2 인스턴스로 분산시켜주기 때문에, 하나의 AWS EC2 인스턴스가 감당해야 하는 트래픽 양을 줄일 수 있다.

### 구성 정보

#### 체계(Scheme)

외부로부터의 접근을 허용할 것인지, 아니면 내부적으로만 사용할 것인지를 나타낸다.

#### 가용 영역(Network Mapping)

서비스되는 존을 나타내며, 일반적으로 2개 이상의 가용 영역을 사용하도록 설정한다.

### 대상 그룹

AWS ELB와 연결되어 로드 밸런싱이 일어날 때, 트래픽을 분담하여 처리할 AWS EC2 인스턴스 목록을 설정할 수 있는 기능이다.

#### AWS EC2 인스턴스 수동 추가

AWS EC2 인스턴스를 직접 생성하여 AWS ELB에 연결된 대상 그룹에 추가하면, 추가된 AWS EC2 인스턴스가 자동으로 트래픽을 분담하여 처리한다.

#### AWS EC2 인스턴스 자동 추가

AWS Auto Scaling 서비스를 이용하면 설정한 조건에 따라 AWS EC2 인스턴스 수가 자동으로 조정되도록 할 수 있는데, 이를 AWS ELB에 연결하면 자동으로 추가된 AWS EC2 인스턴스도 트래픽을 분담하여 처리하도록 만들 수 있다.

## Java

### 변수

어떤 데이터를 저장하기 위한 메모리 공간에 이름을 붙여둔 것으로, 언제든지 할당한 값을 변경할 수 있다.

``` java
// int 자료형 변수 선언
int num = 5;

System.out.println(num);

// String 자료형 변수 선언
String str = "Hello, world!";

System.out.println(str);

// 변수에 할당한 값 변경
str = "Bye, world!";

System.out.println(str);
```

``` bash
[출력]
5
Hello, world!
Bye, world!
```

### 상수

변수와 달리 초기에 할당한 값을 나중에 변경할 수 없다.

``` java
final int MAX = 10;

System.out.println(MAX);
```

``` bash
[출력]
10
```

##### 배열

동일한 자료형의 데이터를 연속된 공간에 저장하기 위한 자료 구조이다.

``` java

// String 배열 자료형 변수 선언
String[] array3 = new String[5];

System.out.println(Arrays.toString(array3));

// 인덱스를 통해 배열 값에 접근

array5[0] = 5;

System.out.println(array6.length);
```

### 조건문

조건의 결과에 따라 프로그램 실행 흐름을 분기하기 위한 구문이다.

#### if-else 문

`if` 조건식이 참이라면 `if` 뒤의 명령문을 실행하고, 거짓이라면 `else` 뒤의 명령문을 실행한다.

``` java
int age = 20;

if (age < 10)
    System.out.println("10살 미만");
else if (age < 20) {
    System.out.println("20살 미만");
}
else {
    if (age == 20) {
        System.out.println("20살");
    }
    else if (age == 21) {
        System.out.println("21살");
    }
    else {
        System.out.println("22살 이상");
    }
}
```

``` bash
[출력]
20살
```

#### switch 문

`switch` 뒤에 오는 변수의 값에 따라 실행을 분기하는 구문으로, 변수의 값과 일치하는 `case` 뒤에 오는 모든 명령문을 실행한다.

``` java
int dayOfWeek = 0;

switch (dayOfWeek) {
    case 0:
        System.out.println("월요일");

    case 1:
        System.out.println("화요일");

    case 2:
        System.out.println("수요일");
        break;

    case 3:
        System.out.println("목요일");
        break;

    case 4:
        System.out.println("금요일");
        break;

    case 5:
        System.out.println("토요일");
        break;

    case 6:
        System.out.println("일요일");
        break;

    default:
        System.out.println("오류");
}
```

``` bash
[출력]
월요일
화요일
수요일
```

### 반복문

특정 명령문을 여러 차례 반복하여 실행하기 위한 구문이다.

#### for 문

초기식, 조건식, 증감식 구조로 구성되며, 조건식을 만족하는 동안 뒤에 오는 명령문을 반복하여 실행한다.

``` java
for (int index = 0; index < 3; index++) {
    System.out.println(index);
}
```

``` bash
[출력]
0
1
2
```

#### for-each 문

배열 또는 `java.lang.Iterable`을 상속받은 클래스의 요소를 순차적으로 하나씩 순회한다.

``` java
int[] array = { 1, 2, 3 };

for (int x : array) {
    System.out.println(x);
}
```

``` bash
[출력]
1
2
3
```

#### while 문

조건식이 참인 동안 뒤에 오는 명령문을 반복해서 실행한다.

``` java
int index = 0;

while (index < 3) {
    System.out.println(index);
    index++;
}
```

``` bash
[출력]
0
1
2
```

#### do-while 문

while 문처럼 조건식이 참인 동안 `do` 뒤에 오는 명령문을 반복하여 실행하나, 조건의 만족 여부와 상관없이 무조건 `do` 뒤에 오는 명령문을 1회 수행한다.

``` java
int index = 3;

do {
    System.out.println(index);
    index++;
} while(index < 3);
```

``` bash
[출력]
3
```

#### 반복문 제어

반복문의 뒤에 오는 명령문을 수행하는 도중 반복을 제어하기 위한 구문이다.

##### continue

명령문 수행 도중 `continue`를 만나면, 뒤에 오는 명령문을 건너뛰고 다음 반복을 수행한다.

``` java
for (int index = 0; index < 5; index++) {
    if (index == 2) {
        continue;
    }

    System.out.println(index);
}
```

``` bash
[출력]
0
1
3
4
```

##### break

명령문 수행 도중 `break`를 만나면, 뒤에 오는 명령문을 건너뛰고 반복을 즉시 종료한다.

``` java
for (int index = 0; index < 5; index++) {
    if (index == 2) {
        break;
    }

    System.out.println(index);
}
```

``` bash
[출력]
0
1
```

### 클래스(Class)

어떤 대상이 가질 수 있는 상태와 행위의 집합을 정의해놓은 것으로, 클래스에서 상태와 행위는 각각 필드와 메서드라는 형태로 표현된다.

``` java
class Car {
    private int speed = 0;

    public void go() {
        speed += 10;
    }

    public void back() {
        speed -= 10;
    }
}
```

#### 인스턴스(Instance)

클래스를 사용하기 위해 메모리 상에 할당한 것을 인스턴스라고 부른다.

``` java
Car car1 = new Car();
Car car2 = new Car();
```

#### 생성자(Constructor)

인스턴스가 생성될 때 사용되는 인스턴스 초기화 메서드로, 해당 클래스의 인스턴스가 메모리에 할당되는 시점에 자동으로 호출된다. 만약 클래스 내에 아무런 생성자도 정의되어 있지 않다면, Java 컴파일러에서 아무런 작업도 하지 않는 기본 생성자를 자동으로 생성해준다.

``` java
class Phone {
    private String model;
    private String color;

    Phone(String model, String color) {
        this.model = model;
        this.color = color;
    }
}

class Main {
    public static void main(String[] args) {
        Phone galaxy = new Phone("Galaxy S10", "Black");
    }
}
```

### 추상 클래스(Abstract Class)

시그니처만 선언되어 있는 추상 메서드를 하나 이상 가지고 있는 클래스를 추상 클래스라고 하며, 추상 메서드를 모두 정의한 서브 클래스를 통해서만 인스턴스를 생성할 수 있다.

``` java
abstract class Bird {
    private int x;
    private int y;
    private int z;

    abstract boolean isFlyable(int z);

    public void fly(int x, int y, int z) {
        this.x = x;
        this.y = y;

        if (isFlyable(z) == true) {
            this.z = z;
        }
        else {
            System.out.println("날 수 없는 높이");
        }
    }

    public void displayLocation() {
        System.out.println("(" + x + ", " + y + ", " + z + ")");
    }
}

class Pigeon extends Bird {
    @Override
    boolean isFlyable(int z) {
        return z < 100;
    }
}

class Main {
    public static void main(String[] args) {
        Bird pigeon = new Pigeon();

        pigeon.fly(10, 20, 30);
        pigeon.displayLocation();

        pigeon.fly(50, 50, 150);
        pigeon.displayLocation();
    }
}
```

``` bash
[출력]
(10, 20, 30)
날 수 없는 높이
(50, 50, 30)
```

### 인터페이스(Interface)

클래스와 달리 인터페이스는 상수와 추상 메서드로만 구성될 수 있으며, 추상 클래스와 마찬가지로 추상 메서드를 모두 정의한 서브 클래스를 통해서만 인스턴스를 생성할 수 있다. 또한 모든 필드는 상수이기 때문에 `public static final`로 정의되어야 하고, 모든 메서드는 추상 메서드이기 때문에 `public abstract`로 선언되어야 한다. 만약 인터페이스에서 필드와 메서드를 만들 때 이러한 접근 제어자를 생략하고 작성한다면, Java 컴파일러에서 알맞는 접근 제어자를 자동으로 추가해준다.

``` java
interface Bird {
    void fly(int x, int y, int z);
}
```

### 상속(Inheritance)

재사용 또는 확장을 위해 어떤 클래스나 인터페이스의 필드와 메서드를 물려받을 수 있는 기법이다.

#### 클래스 상속

`extends`를 이용하여 슈퍼 클래스에 정의된 필드와 메서드를 서브 클래스에서 물려받을 수 있으며, Java에서는 클래스 다중 상속을 허용하지 않는다.

``` java
class Animal {
    protected String name;

    public void cry() {
        System.out.println(name + " 울음");
    }
}

class Dog extends Animal {
    Dog(String name) {
        this.name = name;
    }

    public void swim() {
        System.out.println(name + " 수영");
    }
}

class Main {
    public static void main(String[] args) {
        Dog dog1 = new Dog("코코");
        dog1.cry();
        dog1.swim();

        Animal dog2 = new Dog("초코");
        dog2.cry();
    }
}
```

``` bash
[출력]
코코 울음
코코 수영
초코 울음
```

#### 인터페이스 상속

`implements`를 이용하여 인터페이스를 상속받을 수 있으며, 클래스와 달리 다중 상속이 가능하다.

``` java
interface Bird {
    void fly(int x, int y, int z);
}

class Pigeon implements Bird {
    private int x;
    private int y;
    private int z;

    @Override
    public void fly(int x, int y, int z) {
        this.x = x;
        this.y = y;

        if (z < 100) {
            this.z = z;
        }
    }
}
```

##### super

서브 클래스에서 슈퍼 클래스에 정의된 필드나 메서드를 사용하기 위한 키워드이다.

``` java
class Super {
    protected int num = 10;
}

class Sub extends Super {
    protected int num = 20;

    public void display() {
        System.out.println(num);
        System.out.println(this.num);
        System.out.println(super.num);
    }
}

class Main {
    public static void main(String[] args) {
        Sub sub = new Sub();
        sub.display();
    }
}
```

``` bash
[출력]
20
20
10
```

### 오버로딩(Overloading)

하나의 클래스 내에서 동일한 이름의 메서드를 여러 개 정의하는 것으로, 오버로딩된 메서드들이 갖는 파라미터의 개수나 자료형이 반드시 달라야 한다.

``` java
class Adder {
    public int add(int x, int y) {
        return x + y;
    }

    public int add(int x, int y, int z) {
        return x + y + z;
    }

    public float add(float x, float y) {
        return x + y;
    }
}
```

### 오버라이딩(Overriding)

슈퍼 클래스에 정의된 메서드를 서브 클래스에서 재정의하는 것으로, 오버라이딩된 메서드의 시그니처는 슈퍼 클래스에서 정의한 것과 동일해야 한다.

``` java
class Animal {
    public void cry() {
        System.out.println("울음");
    }
}

class Dog extends Animal {
    @Override
    public void cry() {
        System.out.println("짖음");
    }
}

class Main {
    public static void main(String[] args) {
        Animal dog = new Dog();
        dog.cry();
    }
}
```

``` bash
[출력]
짖음
```

### 패키지(Package)

서로 관련성을 갖는 클래스나 인터페이스를 디렉터리 단위로 묶어 놓은 것을 의미한다.

#### 접근 제어자(Access Modifier)

외부에서 클래스에 정의한 필드나 메서드에 접근하는 것을 제한하는 역할을 수행한다.

| 접근 제어자 | 같은 패키지의 클래스 | 같은 패키지의 서브 클래스 | 다른 패키지의 서브 클래스 | 다른 패키지의 클래스 |
| --- | --- | --- | --- | --- |
| public | O | O | O | O |
| protected | O | O | O | X |
| default | O | O | X | X |
| private | X | X | X | X |

### 예외 처리(Exception Handling)

프로그램 실행 도중 여러가지 이유에 의해서 예외 상황이 발생할 수 있는데, Java에서는 그러한 상황에 대응할 수 있도록 예외를 처리할 수 있는 방법을 제공해주고 있다.

#### Throwable 클래스

자바에서는 `java.lang.Throwable` 클래스를 상속하여 다룰 수 있는 예외의 종류를 표현하고 있다. 이를 직접적으로 상속받고 있는 서브 클래스로는 프로그램이 종료되어야 할 만큼 심각한 문제를 나타내기 위한 `java.lang.Error` 클래스와 그 밖의 예외를 나타내기 위한 `java.lang.Exception` 클래스가 있다.

#### try-catch-finally 문

`try` 안에 작성된 명령문에서 예외가 발생하면, 앞에 있는 `catch`부터 순차적으로 탐색하여 발생한 예외를 다루고 있는 `catch`에 작성한 명령문을 실행한다. 만약 예외 발생 여부와 상관없이 무조건 실행되어야 하는 명령문이 있다면 필요에 따라 `finally`를 이용하여 처리할 수 있다.

``` java
FileOutputStream file;

try {
    file = new FileOutputStream("text.txt");
    file.write("Hello, world!".getBytes());
    file.flush();
}
catch (FileNotFoundException e) {
    System.out.println("파일 찾기 실패: " + e.getMessage());
}
catch (IOException e) {
    System.out.println("I/O 에러: " + e.getMessage());
}
finally {
    file.close();
}
```

#### try-with-resources 문

`java.lang.AutoCloseable` 인터페이스를 상속받는 클래스의 경우 자원의 반납을 위해 `close()` 메서드를 명시적으로 호출해주어야 하는데, try-with-resources 문을 이용하게 되면 `try`에 작성한 명령문이 수행된 이후 자동으로 `close()` 메서드가 호출된다.

``` java
try (FileOutputStream file = new FileOutputStream("text.txt")) {
    file.write("Hello, world!".getBytes());
    file.flush();
}
catch (IOException e) {
    e.printStackTrace();
}
```

#### throws

어떤 메서드에서 발생한 예외를 해당 메서드를 호출한 곳에서 처리하도록 책임을 전가하기 위해 사용한다.

``` java
void writeFile() throws IOException {
    FileOutputStream file = new FileOutputStream("text.txt");
    file.write("Hello, world!".getBytes());
    file.flush();
    file.close();
}
```

### 컬렉션(Collection)

다수의 데이터를 다루기 위한 자료 구조를 구현해 둔 클래스 집합을 의미한다.

#### List

순서가 있는 데이터 집합을 다루며, 데이터의 중복을 허용한다.

``` java
List<Integer> list = new ArrayList<>();

// List 요소 추가
list.add(1);
list.add(5);
list.add(4);
list.add(11);
list.add(10);

System.out.println(list);

// List 정렬
Collections.sort(list);

System.out.println(list);

// List 크기
System.out.println(list.size());

// 인덱스에 해당하는 List 요소 삭제
list.remove(4);

System.out.println(list);

// for 문을 이용한 List 순회
for (int index = 0; index < list.size(); index++) {
    System.out.println(list.get(index));
}

// for-each 문을 이용한 List 순회
for (int element : list) {
    System.out.println(element);
}
```

``` bash
4

```

#### Set

순서와 무관한 데이터 집합을 다루며, 데이터의 중복을 허용하지 않는다.

``` java
Set<Integer> set = new HashSet<>();

// Set 요소 추가
set.add(1);
set.add(1);
set.add(3);
set.add(2);
set.add(9);
set.add(11);

System.out.println(set);

// Set 요소 한 개 삭제
set.remove(9);

System.out.println(set);

// Set 요소 여러 개 삭제
List<Integer> target = new ArrayList<>();

target.add(3);
target.add(2);

set.removeAll(target);

System.out.println(set);

// Set 요소 포함 여부
System.out.println(set.contains(1));
System.out.println(set.contains(9));

// Set 크기
System.out.println(set.size());

// Set 모든 요소 삭제
set.clear();

System.out.println(set);
```

#### Map

순서와 무관한 키(Key)-값(Value) 쌍으로 이루어진 데이터 집합을 다루며, 키는 중복을 허용하지 않고 값은 중복을 허용한다.

``` java
Map<Integer, String> map = new HashMap<>();

// Map 요소 추가
map.put(1, "Apple");
map.put(2, "Berry");
map.put(5, "Cherry");

System.out.println(map);

// Map 요소 획득
System.out.println(map.get(1));
System.out.println(map.get(5));

// Map 요소 삭제
map.remove(2);

System.out.println(map);

// Map 키 포함 여부
System.out.println(map.containsKey(2));

// Map 값 포함 여부
System.out.println(map.containsValue("Cherry"));

// Map 모든 요소 삭제
map.clear();

System.out.println(map);
```

#### Stack

마지막에 넣은 데이터를 먼저 꺼내는 자료 구조이다.

``` java
Stack<Integer> stack = new Stack<>();

// Stack 요소 추가
stack.push(1);
stack.push(3);
stack.push(5);
stack.push(2);

System.out.println(stack);

// Stack 최상단 요소 획득
System.out.println(stack.peek());

// Stack 최상단 요소 삭제
int top = stack.pop();

System.out.println(top);
System.out.println(stack);

// Stack 크기
System.out.println(stack.size());

// Stack 요소 포함 여부
System.out.println(stack.contains(1));

// Stack 비어있음 여부
System.out.println(stack.isEmpty());
```

#### Queue

먼저 넣은 데이터를 먼저 꺼내는 자료 구조이다.

``` java
Queue<Integer> queue = new LinkedList<>();

// Queue 요소 추가
queue.add(1);
queue.add(3);
queue.add(5);

System.out.println(queue);

// Queue 첫 번째 요소 획득
System.out.println(queue.peek());

// Queue 첫 번째 요소 삭제
int front = queue.poll();

System.out.println(front);
System.out.println(queue);
```

#### Deque

양 끝에서 데이터를 넣고 꺼낼 수 있는 자료 구조이다.

``` java
Deque<Integer> deque = new ArrayDeque<>();

// Deque 앞쪽에 요소 추가
deque.addFirst(1);
deque.addFirst(2);
deque.addFirst(3);

System.out.println(deque);

// Deque 뒤쪽에 요소 추가
deque.addLast(0);

System.out.println(deque);

// Deque 크기
System.out.println(deque.size());

// Deque 앞쪽의 요소 삭제
int front = deque.removeFirst();

System.out.println(front);
System.out.println(deque);

// Deque 뒤쪽의 요소 삭제
int rear = deque.removeLast();

System.out.println(rear);
System.out.println(deque);

// Deque 앞쪽의 요소 획득
System.out.println(deque.peekFirst());

// Deque 뒤쪽의 요소 획득
System.out.println(deque.peekLast());
```

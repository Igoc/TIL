## Docker

소프트웨어를 컨테이너라는 표준화된 유닛으로 패키징하여, 애플리케이션을 신속하게 구축하고 배포할 수 있도록 만들어주는 소프트웨어 플랫폼이다.

### 이미지(Image)

애플리케이션을 실행하는 데 필요한 소스 코드, 라이브러리, 도구 등을 포함하고 있는 파일로 불변성을 가지고 있다.

#### Dockerfile을 통한 이미지 생성

``` bash
docker build -t <이미지명>:<태그> <Dockerfile 경로>
```

#### 이미지 목록

``` bash
docker images
```

#### 이미지 삭제

``` bash
docker rmi <이미지명>
```

#### 이미지 복제

``` bash
docker tag <원본 이미지명>:<원본 태그> <복제 이미지명>:<복제 태그>
```

### 컨테이너(Container)

이미지를 실행할 수 있는 형태로 만들어 둔 인스턴스이다.

#### 컨테이너 생성

``` bash
# 도커 컨테이너를 생성함과 동시에 실행
docker run --name <컨테이너명> -p <호스트 포트>:<컨테이너 포트> -d <이미지명>:<이미지 태그>

# host 네트워크 방식으로 도커 컨테이너를 생성함과 동시에 실행
docker run --name <컨테이너명> --net=host -d <이미지명>:<이미지 태그>
```

#### 컨테이너 실행 및 종료

``` bash
# 컨테이너 실행
docker start <컨테이너명>

# 컨테이너 종료
docker stop <컨테이너명>
```

#### 컨테이너 목록

``` bash
# 실행 중인 컨테이너 목록 출력
docker ps

# 모든 컨테이너 목록 출력
docker ps -a
```

#### 컨테이너 삭제

``` bash
docker rm <컨테이너명>
```

#### 컨테이너 로그 조회

``` bash
docker logs -f <컨테이너명>
```

#### 컨테이너에서 호스트 주소에 연결

컨테이너마다 독립적인 가상 네트워크를 가지고 있기 때문에, 호스트 주소에 연결하려면 `localhost` 대신 `host.docker.internal`를 이용해야 한다.

### 레지스트리(Registry)

이미지를 배포하고 관리할 수 있는 저장소이다.

#### 로그인

``` bash
docker login
```

#### Push

레지스트리에 Push하기 위해서는 레지스트리의 이름과 이미지의 이름이 동일해야 한다.

``` bash
docker push <레지스트리명>:<태그>
```

#### Pull

``` bash
docker pull <레지스트리명>:<태그>
```
